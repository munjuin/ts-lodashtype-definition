# 🚀 TS-Lodash Type Definition Mastery

오픈소스 라이브러리인 **Lodash**의 핵심 유틸리티 함수들을 TypeScript로 재설계하며, 고수준 타입 시스템의 활용법과 실무적인 트러블슈팅 경험을 기록한 프로젝트입니다.

## 📌 주요 설계 의도 및 핵심 기술

### 1. `groupBy` (함수 오버로딩 및 제네릭)

- **설계 의도**: 사용자가 데이터를 분류할 때 '콜백 함수' 또는 '객체의 키(문자열)'를 인자로 전달할 수 있는 유연성을 제공합니다.
- **핵심 기술**:
  - **Function Overloading**: 서로 다른 입력 형태에 대해 최적화된 타입 추론과 자동 완성을 제공합니다.
  - **`K extends keyof T`**: 객체에 존재하지 않는 속성 이름을 입력할 경우 컴파일 단계에서 차단하여 안정성을 확보했습니다.

### 2. `debounce` (함수 시그니처 보존 및 기능 확장)

- **설계 의도**: 데바운싱된 함수가 원본 함수의 매개변수 정보를 잃지 않으면서, 추가적인 메서드(`.cancel()`, `.flush()`)를 호출할 수 있도록 설계했습니다.
- **핵심 기술**:
  - **Generic `F`**: 원본 함수의 인자와 반환 타입을 통째로 캡처하여 복제하는 기술을 적용했습니다.
  - **Intersection Types (`&`)**: 함수 타입과 인터페이스를 결합하여 '함수이면서 동시에 객체인' 하이브리드 타입을 구현했습니다.

### 3. `get` (재귀적 문자열 경로 타입 추론)

- **설계 의도**: `'user.profile.name'`과 같이 점(`.`)으로 구분된 문자열 경로를 해석하여 깊은 곳에 위치한 데이터의 타입을 정확히 찾아냅니다.
- **핵심 기술**:
  - **Template Literal Types & `infer`**: 문자열을 동적으로 파싱하여 왼쪽과 오른쪽으로 분리하는 재귀적 설계를 도입했습니다.
  - **Recursive Conditional Types**: 경로의 깊이에 상관없이 최종 도달한 속성의 타입을 정확히 반환합니다.

## 🔍 주요 트러블슈팅 경험

### 1. 런타임과 컴파일 타임의 간극 해결 (`is not a function`)

- **문제**: `.d.ts` 파일에 타입만 선언하고 구현체가 없어 발생한 실행 에러를 해결했습니다.
- **해결**: 타입 설계와 실제 실행 로직을 통합한 `.ts` 모듈 구조로 변경하고, `ts-node` 환경에서 경로 별칭(`Path Alias`)을 인식하도록 `tsconfig-paths` 설정을 최적화했습니다.

### 2. 패키지 실행 환경 최적화

- **문제**: `package.json`의 JSON 문법 오류 및 실행 명령어 복잡도 증가 문제를 해결했습니다.
- **해결**: 실행 스크립트를 자동화하여 `npm start` 명령어 하나로 타입 체크부터 경로 매핑, 실행까지 한 번에 수행되도록 DX(개발자 경험)를 개선했습니다.
